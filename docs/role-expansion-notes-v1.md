# NERV Role Expansion Notes v1

## 目标

这份文档用来定义：

- 哪些能力扩张应该回写到角色边界
- 哪些能力不应该进入 SOUL
- 各 Agent 在 `domain expansion` 中的稳定增量是什么

原则：

- **不新增 Agent**
- **不重写人格**
- **只补稳定职责边界**
- **行业细节优先进入 skill pack 和 workflow**

## 回写到 SOUL 的标准

只有满足以下条件，才应该回写到 SOUL：

1. 已在真实 workflow 中跑过
2. 主责边界稳定
3. 不会把 prompt 继续膨胀成行业手册
4. 能用一句话解释清楚“它主责什么，不主责什么”

## 不回写到 SOUL 的内容

- 平台操作步骤
- 具体 campaign 玩法
- 每一个 workflow 的节点细节
- 需要经常迭代的策略模板

这些内容应该留在：

- `skill-pack-registry-v1`
- `workflow-template-catalog-v1`
- 或未来的具体 skill 文档中
